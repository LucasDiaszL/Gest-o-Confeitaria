## 🛠️ Evidências de Desenvolvimento Individual

Para fins de avaliação e auditoria acadêmica, as contribuições individuais podem ser validadas diretamente no repositório através dos seguintes artefatos:

### 1. Histórico de Commits e Pull Requests
* **Módulo de Autenticação e Segurança (Lucas Jorge):** Commits referentes à integração de rotas protegidas e validação de token OTP via Supabase.
* **Módulo de Estoque e Caixa (Lucas Jorge):** Commits contendo a estrutura de componentes React (Tabelas, Modais de Lançamento) e hooks customizados.
* **Ficha Técnica e Regras (Letícia Rohod / Lucas Jorge):** Commits de alinhamento das fórmulas de CMV do backend com a interface visual.

### 2. Gestão de Issues e Tarefas (GitHub Projects / Kanban)
O fluxo de trabalho foi gerenciado via metodologia ágil Scrum/Kanban. As atribuições individuais foram documentadas através de:
* **Assinatura de Issues:** Cada funcionalidade (ex: *#01 - Cadastro de Insumos*, *#02 - Fluxo 2FA*) possui um integrante atribuído (`Assignee`).
* **Critérios de Aceitação:** Descritos detalhadamente nas issues para guiar os testes de caixa preta realizados pela equipe.

### 3. Registros de Participação (Dailies e Reuniões)
* **Atas de Alinhamento:** Registros de reuniões semanais para validação do design e das regras de negócio propostas pela gestão de produto antes do início do desenvolvimento de cada sprint.

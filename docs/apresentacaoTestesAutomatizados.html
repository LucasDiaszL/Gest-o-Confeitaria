Ferramenta 1 · Vitest

O que é um Teste Unitário?

Testa funções e componentes isoladamente, sem banco de dados real

🎯 Como funciona

Substitui o banco de dados por um mock (objeto fake). A função é chamada com dados controlados e o teste verifica se o resultado é o esperado — sem internet, sem Supabase.

⚡ Por que usar

Os 67 testes do projeto rodam em menos de 3 segundos. Cada função tem cenários testados: dado válido, dado inválido, erro do banco.

📦 Onde está no projeto

8 arquivos de teste cobrindo serviços, hooks e componentes:

insumos.test.js produtos.test.js receitas.test.js vendas.test.js useInsumos.test.js Estoque.test.jsx Sidebar.test.jsx SimuladorReajuste.test.jsx

Padrão AAA: Arrange → Act → Assert

Prepara os dados · Executa a função · Verifica o resultado

Ferramenta 2 · Cypress

O que é um Teste E2E?

End-to-End — testa a aplicação real no navegador, simulando um usuário de verdade

🌐 Como funciona

O Cypress abre o navegador, faz login com credenciais reais, clica em botões, preenche formulários e valida o que aparece na tela — exatamente como um usuário faria.

🔗 Por que usar

Garante que o fluxo completo funciona de ponta a ponta: do clique na interface até a resposta do banco de dados. Pega bugs que testes unitários não conseguem detectar.

📦 Onde está no projeto

cypress/e2e/venda.cy.js cypress/e2e/relatorios.cy.js cypress.config.cjs

venda.cy.js

Faz uma venda e verifica que o estoque do insumo diminuiu automaticamente

relatorios.cy.js

Navega pela tela de relatórios e filtra por Hoje, 7 dias e 30 dias

Comparação

Unitário vs E2E

Cada ferramenta tem um papel diferente — as duas se complementam

Característica Vitest · Unitário Cypress · E2E
O que testa Funções e componentes isolados Fluxo completo no navegador
Banco de dados ✗ Mock (fake) ✓ Supabase real
Precisa do app rodando ✗ Não ✓ Sim (localhost:5173)
Velocidade ⚡ ~3 segundos (67 testes) 🐢 ~30–60 segundos
Simula usuário real ✗ Não ✓ Clica, digita, navega
Detecta bug de lógica ✓ Sim ✓ Sim
Detecta bug de integração ✗ Não (usa mock) ✓ Sim
Como Rodar

Comandos do Projeto

Todos os comandos reais disponíveis no package.json

🧪

Vitest — Testes Unitários

Não precisa do servidor rodando

Watch mode — reexecuta ao salvar (desenvolvimento)

npm test

Execução única e encerra

npm run test:run

Mostra cada teste individualmente

npm run test:run -- --reporter=verbose

Relatório de cobertura de código

npm run test:coverage

Só um arquivo específico

npm run test:run -- src/services/insumos.test.js
🌐

Cypress — Testes E2E

Precisa do servidor rodando primeiro

1. Sobe a aplicação (terminal 1)

npm run dev

2a. Interface visual — melhor para apresentação (terminal 2)

npx cypress open

2b. Modo headless — só no terminal

npx cypress run

Rodar um arquivo específico

npx cypress run --spec "cypress/e2e/venda.cy.js"
